## 🚀 Major Features
- Added dual AI provider support: translation can now run through `GPT4Free` or `DeepSeek`, with a provider toggle directly in the main workflow.
- Introduced advanced AI configuration in settings, including provider/model selection, API keys, temperature, and per-provider batch sizes.
- Added dedicated translation modes for `Single File` and `Resource Pack`, expanding beyond the previous mod-centric workflows.
- Added raw text translation/editing alongside the table-based translation view, making it possible to work with full file content when structured parsing is not ideal.
- Added dedicated log/progress UX for GPT4Free installation and startup, including a separate log window and live streaming output during translation.
- Added a dependency onboarding flow that checks for Python and Git, and guides installation before GPT4Free is used.

## ⚡ Improvements
- Translation requests are now streamed, retried, and protected with timeout and rate-limit handling, making long-running translation sessions much more resilient.
- Batch processing is more flexible and configurable, with separate tuning for GPT4Free and DeepSeek instead of a single fixed batch size.
- Saving is more robust: translations are persisted incrementally during processing instead of only at the very end, reducing the risk of losing progress on cancellation or failure.
- Added caching of loaded localization documents and raw content snapshots, which improves responsiveness when switching between files.
- Tree selection is smarter, with tri-state behavior, automatic locale-aware targeting, and commands for selecting nodes missing the target locale.
- The UI now supports richer interaction patterns such as row click/drag selection, contextual copy actions, highlighted search results, and an expandable translation console.

## 🛠 Refactoring
- Replaced several large, monolithic classes with a more modular structure: `MainViewModel`, `SettingsViewModel`, and translation logic are now split into focused partials and supporting services.
- Introduced a composition/bootstrap layer with explicit interfaces, moving the app away from hardwired concrete dependencies toward a cleaner service-oriented design.
- Reworked the localization pipeline around dedicated components such as `LocalizationDocumentStore`, `LocalizationContentParser`, `TranslationOrchestrator`, and `LocalizationArchiveWriter`.
- Split AI request handling by provider, with dedicated GPT4Free and DeepSeek implementations instead of a single hardcoded request path.
- Reorganized services into clearer domains such as `Ai`, `Core`, and `Translation`, improving maintainability and separation of concerns.
- Replaced legacy save/loading helpers like `LocalizationSaveManager`, `LocalizationStringManager`, `SnbtManager`, and the old flat service layer with more specialized, composable components.
- Moved the main window into a proper `Views/MainWindow` feature folder and expanded reusable behaviors, converters, and styles.

## 🐞 Fixes
- Fixed startup/shutdown lifecycle handling by explicitly creating the main window through the app bootstrapper and disposing GPT4Free services on exit.
- Fixed provider-related failure cases by validating DeepSeek API configuration before translation begins.
- Improved handling of stalled, rate-limited, or partially failed GPT4Free responses so translation no longer depends on a single fragile request path.
- Improved save behavior for Patchouli, quest, and archive outputs through pre-seeding and mode-specific archive writing.
- Reduced the chance of incorrect selection state and missed parent/child updates in the translation tree through proper recursive and parent-aware checkbox logic.

## 📦 Other Changes
- Removed the old external `install_gpt4free.bat` and `run_gpt4free.bat` workflow in favor of integrated in-app setup and execution.
- Expanded localized resources, tooltips, and settings descriptions across supported UI languages.
- Refreshed README files to document the new providers, new translation modes, setup requirements, and usage guidance.
- Added new styles, icons, converters, and small UI utilities to support the more advanced interface.
